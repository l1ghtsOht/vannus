Threat Mitigation Matrix
🌊
Session Flooding BLOCKED
HTTP GET/POST flooding to exhaust orchestration memory and CPU.
↳ Axiom 1 — Edge-level rate limiting, SASE authentication
💣
Payload Inflation BLOCKED
Deep circular GraphQL queries, excessive aliases, batch overloading.
↳ Axiom 2 — RASP AST analysis with dynamic upper limits
💉
Prompt Injection BLOCKED
Direct/indirect injection to manipulate LLM behaviour or jailbreak.
↳ Axiom 2 — Deep runtime inspection of execution paths
🔗
AST Alias-Chaining BLOCKED
Recursive alias definitions hiding malicious logic in dynamic languages.
↳ Axiom 4 — Taint propagation source→sink tracking
⚡
Race Conditions MITIGATED
Concurrent writes or file lock bypasses to corrupt persistent state.
↳ Axiom 5 — Strict file-locking + bounded write queues
🤖
Hallucination Cascades MITIGATED
Incorrect LLM output contaminating downstream reasoning tasks.
↳ PRISM Critic — evaluates outputs against GraphRAG evidence
Data Flow — Query Lifecycle
Step 1
Outer Perimeter (Moat)
Validate CORS origin, assess rate parameters. If within threshold, pass to interior.
Step 2
Payload Inspection (Sally Port)
RASP intercepts — inspects for jailbreak heuristics, encoded manipulation, inflation.
Step 3
Identity Verification
Evaluate cryptographic SASE token. Confirm AuthZ access. Map to Workflow layer.
Step 4
AST Taint Analysis
Map data flow source→sink. Detect alias-chaining, unsafe deserialization.
Step 5
PRISM Orchestration
Analyzer decomposes query → Selector filters → Critic validates → Adder completes.
Step 6
Hybrid GraphRAG Retrieval
Vector search + graph traversal. Context aggregation for layered insights.
Step 7
Selection & Critique
Filter outdated context, verify claims against source data, iterate if needed.
Step 8
Persistence & Safeguards
Bounded queue → file-lock acquisition → atomic write → lock release.
Step 9
Egress
Verified response transmitted back through outbound Presentation port.
